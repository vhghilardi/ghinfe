Introdução

A GHINFE é uma biblioteca modular para emissão de documentos fiscais eletrônicos no Brasil. A API foi pensada de forma semelhante ao ACBr: você monta um objeto JavaScript com cabeçalho, itens, frete e pagamentos; a biblioteca gera o XML oficial, assina com certificado A1, envia via SOAP para a SEFAZ e trata o retorno.

Objeto JS → XML NF-e → Assinatura A1 → SOAP SEFAZ → Protocolo / Rejeição

Instalação

git clone <repo> ghinfe
cd ghinfe
npm install
cp .env.example .env
npm start          # API + Swagger em http://localhost:3333
npm run example:nfe

Requisitos: Node.js 18+, Redis (para filas Bull).

Arquitetura

Cada responsabilidade está em um serviço isolado:

ServiçoResponsabilidade
NFeXmlServiceGera XML layout SEFAZ 4.00
NFeSignServiceAssina infNFe com certificado .pfx
NFeSoapServiceEnvia SOAP e interpreta cStat
NFeServiceOrquestra o fluxo completo
NFeQueueProcessamento assíncrono via Bull/Redis

Objeto de domínio

Monte o documento fiscal como um objeto JavaScript plano:

const documento = {
  chaveAcesso: '35260712345678000199550010000000011000000011', // 44 dígitos
  ambiente: 2, // 1=Produção, 2=Homologação
  ide: {
    uf: 'SP',
    codigoMunicipio: '3550308',
    serie: 1,
    numero: 1,
    naturezaOperacao: 'VENDA',
  },
  emitente: {
    cnpj: '12345678000199',
    razaoSocial: 'EMPRESA EXEMPLO LTDA',
    ie: '123456789012',
    crt: '3',
    endereco: { /* logradouro, numero, bairro, codigoMunicipio, municipio, uf, cep */ },
  },
  destinatario: { /* cpf ou cnpj, razaoSocial, endereco */ },
  itens: [{
    codigo: '001',
    descricao: 'PRODUTO TESTE',
    ncm: '84713012',
    cfop: '5102',
    unidade: 'UN',
    quantidade: 1,
    valorUnitario: 100,
    valorTotal: 100,
    cst: '00',
    aliquotaIcms: 18,
    valorIcms: 18,
  }],
  frete: { modalidade: 9 }, // 9 = sem frete
  pagamentos: [{ forma: '01', valor: 100 }],
};

Geração de XML

import { NFeService } from 'ghinfe';

const service = new NFeService();
const xml = service.gerarXml(documento);
// Retorna string XML UTF-8, layout NF-e 4.00, sem assinatura

NFeXmlBuilder

O builder monta as seções ide, emit, dest, det, total, transp e pag conforme o Manual de Orientação do Contribuinte.

Assinatura digital (A1)

import fs from 'fs';
import { NFeService } from 'ghinfe';

const service = new NFeService();
const pfx = fs.readFileSync('./certs/certificado.pfx');

const xmlAssinado = service.gerarXmlAssinado(documento, {
  pfx,
  senha: 'senha-do-certificado',
});

A assinatura usa XML-DSig enveloped no elemento infNFe, com RSA-SHA1 e C14N, conforme exigido pela SEFAZ.

import { NFeSignService } from 'ghinfe';

const signService = new NFeSignService();
const info = signService.obterInfoCertificado(pfx, senha);
console.log(info.validoAte); // data de expiração

Envio SEFAZ

Modo mock (desenvolvimento)

const service = new NFeService({ mock: true, mockScenario: 'autorizado' });
const resultado = await service.emitir(documento, { pfx, senha });

console.log(resultado.retorno.cStat);    // '100'
console.log(resultado.retorno.protocolo); // número do protocolo

Cenários mock disponíveis: autorizado, duplicidade, schema, certificado, paralisado.

Modo produção/homologação real

import { SEFAZ_URLS } from 'ghinfe';

const service = new NFeService({
  mock: false,
  sefazEndpoint: SEFAZ_URLS.SP.autorizacao,
});

Tratamento de erros SEFAZ

import { SefazError } from 'ghinfe';

try {
  await service.enviar(xmlAssinado, chaveAcesso);
} catch (error) {
  if (error instanceof SefazError) {
    console.log(error.cStat);   // ex: '204'
    console.log(error.xMotivo); // ex: 'Rejeição: Duplicidade de NF-e'
    console.log(error.isRejeicao()); // true para cStat 2xx
  }
}
cStatSignificado
100Autorizado o uso da NF-e
103Lote recebido com sucesso
104Lote processado
204Rejeição: Duplicidade de NF-e
225Rejeição: Falha no Schema XML
280Rejeição: Certificado Transmissor inválido

Fila Bull / Redis

import { NFeQueue } from 'ghinfe';

const queue = new NFeQueue({
  redis: { host: '127.0.0.1', port: 6379 },
  nfeOptions: { mock: true },
});

await queue.adicionarEmissao({
  documento,
  certificado: { pfx, senha },
});

queue.onCompleted(async (resultado) => {
  // persistir no banco
});

queue.onFailed(async (error, job) => {
  console.error('Job falhou:', job.id, error.message);
});

A fila usa retry exponencial (3 tentativas) e é ideal para picos de emissão sem bloquear a API do seu sistema.

Integração com Sequelize

// models/nota-fiscal.js
export default (sequelize, DataTypes) => {
  return sequelize.define('NotaFiscal', {
    chave_acesso: { type: DataTypes.STRING(44), unique: true },
    xml_nfe: DataTypes.TEXT,
    xml_protocolo: DataTypes.TEXT,
    protocolo: DataTypes.STRING(20),
    status: DataTypes.STRING(3),
    motivo: DataTypes.STRING(255),
    autorizado_em: DataTypes.DATE,
  });
};

// worker
queue.onCompleted(async (resultado) => {
  const payload = service.toPersistencePayload(resultado);
  await NotaFiscal.create(payload);
});

Swagger / API de testes

Com npm start, acesse:

  • /docs — Swagger UI interativo
  • POST /nfe/xml — gera XML a partir de JSON
  • POST /nfe/enviar-mock?cenario=duplicidade — simula envio
  • GET /nfe/cenarios-mock — lista cenários

Teste rápido via curl

curl -X POST http://localhost:3333/nfe/xml \
  -H "Content-Type: application/json" \
  -d @examples/payload-nfe.json
